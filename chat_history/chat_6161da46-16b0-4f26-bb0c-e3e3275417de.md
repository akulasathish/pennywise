To guarantee **100% stability and prevent any payment from ever matching the wrong tenant**, we need to address the only scenario where a mismatch can occur in a fractional system.

### How a Mismatch Can Happen (The Race Condition)
1. **Tenant A** starts checkout for **Rs. 1,000.00**. They get assigned **Rs. 1,000.01**.
2. **Tenant A**'s 10-minute session expires. The system marks it as `expired` and frees up `1,000.01`.
3. **Tenant B** starts checkout and gets assigned **Rs. 1,000.01**.
4. **Tenant A** (who was slow or whose bank took 15 minutes to process) *now* completes their payment of Rs. 1,000.01.
5. The bank sends an SMS for Rs. 1,000.01, and the system matches it to **Tenant B**!

---

### The Solution: 100% Mismatch-Proof Verification
To make this system rock-solid, we should add a **Secondary Verification Factor (UTR/Ref Number)**:

* **What is UTR?**: Every UPI transaction generates a unique **Unique Transaction Reference (UTR)** or **UPI Ref No** (usually 12 digits, e.g. `Ref No: 612849102938`), which is always present in the bank's credit SMS.
* **The Flow**:
  1. The checkout page prompts the tenant: *"Please enter the UPI transaction ID / Ref No after payment."*
  2. The tenant pays, copies the UTR from their GPay/PhonePe screen, and submits it.
  3. When the SMS arrives, the server checks both:
     - **Match 1**: The exact amount matches.
     - **Match 2**: The UTR in the SMS matches the UTR submitted by the tenant.

This reduces the chance of a mismatch to **0%**. Even if multiple sessions expire or share the same amount, a tenant can never claim someone else's payment.

---

### Which Pattern is More Stable?

* **Pattern B (Direct Integration)** is the most stable overall. Since it runs inside your main Property Management System database, you don't have to worry about network communication failures between two separate servers (PMS and PennyWise) failing to confirm a payment.
* **Pattern A (Microservice)** is also highly stable, but only if you ensure the webhook dispatcher (which pings the PMS from PennyWise) has a retry mechanism in case your PMS is temporarily down.

### The Correct Solution: Cooldown & Late-Matching

To achieve **100% mismatch prevention** without relying on UTR numbers, we can implement a **Cooldown Reservation and Late-Matching Mechanism** on the server side:

#### 1. Cooldown Period (Preventing Instant Reuse)
When a tenant's checkout session expires (e.g. after 10 minutes) or is completed, **we do not immediately reuse that decimal amount** (e.g. `Rs. 100.01`). 
We put that decimal amount into a **30-minute cooldown**. 
* During this 30 minutes, no new checkout can be assigned `100.01`.
* This completely eliminates the risk of another user being assigned that amount while a delayed transfer is in progress.

#### 2. Allowing "Expired" Matches
If the bank SMS is delayed by 15 minutes, the session will have expired. But since we put the decimal amount in cooldown, **no one else has it**. 
* When the SMS for `100.01` eventually arrives, the server matches it to the expired request, updates it to `completed`, and successfully credits the original tenant.
* If a payment arrives hours later (past the cooldown), the system logs it as `unmatched` instead of crediting it to a wrong user. The administrator can then easily credit the user manually from the dashboard.

#### 3. SMS Interceptor Broadcast Receiver (`SMSReceiver.kt`)
This class triggers whenever a new SMS is received. It parses the body and schedules the background sync.

```kotlin
package com.example.pennywise

import android.content.BroadcastReceiver
import android.content.Context
import android.content.Intent
import android.telephony.SmsMessage
import androidx.work.OneTimeWorkRequestBuilder
import androidx.work.WorkManager
import java.util.regex.Pattern

class SMSReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action == "android.provider.Telephony.SMS_RECEIVED") {
            val bundle = intent.extras
            if (bundle != null) {
                val pdus = bundle["pdus"] as Array<*>? ?: return
                for (pdu in pdus) {
                    val message = SmsMessage.createFromPdu(pdu as ByteArray)
                    val sender = message.originatingAddress ?: "UNKNOWN"
                    val body = message.messageBody ?: ""

                    // 1. Filter keywords (e.g. checks if text contains "credited" or "received")
                    val normalized = body.lowercase().replace(",", "")
                    if (normalized.contains("credited") || normalized.contains("received") || normalized.contains("deposited")) {
                        
                        // 2. Parse exact amount using regular expressions
                        val amount = parseAmount(normalized)

                        // 3. Save to local SQLite cache
                        val db = DatabaseHelper(context)
                        db.insertSms(sender, body, amount)

                        // 4. Trigger resilient background sync to Supabase
                        val syncRequest = OneTimeWorkRequestBuilder<SyncWorker>().build()
                        WorkManager.getInstance(context).enqueue(syncRequest)
                    }
                }
            }
        }
    }

    private fun parseAmount(text: String): Double? {
        val patterns = arrayOf(
            Pattern.compile("(?:rs\\.?|inr|rupees)\\s*([0-9]+(?:\\.[0-9]{1,2})?)", Pattern.CASE_INSENSITIVE),
            Pattern.compile("credited\\s+(?:with|by|of)?\\s*(?:rs\\.?|inr)?\\s*([0-9]+(?:\\.[0-9]{1,2})?)", Pattern.CASE_INSENSITIVE),
            Pattern.compile("received\\s+(?:rs\\.?|inr)?\\s*([0-9]+(?:\\.[0-9]{1,2})?)", Pattern.CASE_INSENSITIVE),
            Pattern.compile("deposited\\s+(?:rs\\.?|inr)?\\s*([0-9]+(?:\\.[0-9]{1,2})?)", Pattern.CASE_INSENSITIVE)
        )
        for (pattern in patterns) {
            val matcher = pattern.matcher(text)
            if (matcher.find()) {
                try {
                    return matcher.group(1)?.toDouble()
                } catch (e: Exception) {
                    // Ignore parsing error
                }
            }
        }
        return null
    }
}
```

#### 4. Resilient Network Sync Worker (`SyncWorker.kt`)
This handles sending cached transactions from the SQLite database to your Supabase instance, retrying automatically with exponential backoff if the phone goes offline.

```kotlin
package com.example.pennywise

import android.content.Context
import android.database.Cursor
import androidx.work.CoroutineWorker
import androidx.work.WorkerParameters
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.OkHttpClient
import okhttp3.Request
import okhttp3.RequestBody.Companion.toRequestBody
import org.json.JSONObject

class SyncWorker(appContext: Context, workerParams: WorkerParameters) :
    CoroutineWorker(appContext, workerParams) {

    // IMPORTANT: Set your Supabase API details
    private val supabaseUrl = "YOUR_SUPABASE_PROJECT_URL"
    private val supabaseAnonKey = "YOUR_SUPABASE_ANON_KEY"

    override suspend fun doWork(): Result {
        val dbHelper = DatabaseHelper(applicationContext)
        val db = dbHelper.writableDatabase

        val cursor: Cursor = db.rawQuery(
            "SELECT * FROM ${DatabaseHelper.TABLE_SMS} WHERE ${DatabaseHelper.COLUMN_SYNCED} = 0", 
            null
        )

        val client = OkHttpClient()
        val mediaType = "application/json; charset=utf-8".toMediaType()

        var hasFailure = false

        if (cursor.moveToFirst()) {
            do {
                val id = cursor.getInt(cursor.getColumnIndexOrThrow(DatabaseHelper.COLUMN_ID))
                val sender = cursor.getString(cursor.getColumnIndexOrThrow(DatabaseHelper.COLUMN_SENDER))
                val body = cursor.getString(cursor.getColumnIndexOrThrow(DatabaseHelper.COLUMN_BODY))
                val amount = cursor.getDouble(cursor.getColumnIndexOrThrow(DatabaseHelper.COLUMN_AMOUNT))

                // Prepare database payload for Supabase REST API insert
                val jsonPayload = JSONObject().apply {
                    put("sender", sender)
                    put("body", body)
                    put("parsed_amount", amount)
                    put("status", "unmatched") // Database triggers handle matching
                }

                val requestBody = jsonPayload.toString().toRequestBody(mediaType)
                val request = Request.Builder()
                    .url("$supabaseUrl/rest/v1/pennywise_sms_logs")
                    .post(requestBody)
                    .addHeader("apikey", supabaseAnonKey)
                    .addHeader("Authorization", "Bearer $supabaseAnonKey")
                    .addHeader("Content-Type", "application/json")
                    .build()

                try {
                    val response = client.newCall(request).execute()
                    if (response.isSuccessful) {
                        // Mark as synced locally
                        db.execSQL("UPDATE ${DatabaseHelper.TABLE_SMS} SET ${DatabaseHelper.COLUMN_SYNCED} = 1 WHERE ${DatabaseHelper.COLUMN_ID} = $id")
                    } else {
                        hasFailure = true
                    }
                    response.close()
                } catch (e: Exception) {
                    hasFailure = true
                }
            } while (cursor.moveToNext())
        }
        cursor.close()

        return if (hasFailure) {
            Result.retry() // Tells Android system to automatically retry later
        } else {
            Result.success()
        }
    }
}
```
